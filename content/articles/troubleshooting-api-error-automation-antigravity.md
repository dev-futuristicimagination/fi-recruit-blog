ある日の朝8時15分。私のSlackは、文字通り真っ赤に染まっていた。

「Buffer API: Unauthorized (401)」
「YouTube Data API: Quota Exceeded / Token Invalid」
「SEO Rewrite Pipeline: Gemini API connection failed」

自社で運営している11の自動化メディアや、クライアントに提供しているGemini APIのパイプラインなど、10以上の稼働中プロジェクトで一斉にエラーが検知された。ソーシャルメディアへの自動投稿、動画データの収集、検索クエリに基づいたSEO自動リライト。これらを裏側で支えていたトークンやAPIキーの認証ロジックが、ある特定の依存ライブラリのアップデートとレートリミットの変更によって、ドミノ倒しのように全滅したのだ。

普通の会社なら、ここで緊急ミーティングを開き、担当エンジニアに調査を依頼し、原因を特定して、夕方ごろにようやく修正プルリクエストが作られるのだろう。

しかし、Futuristic Imagination（FI）にその選択肢はない。私たちは「自動化」を極限まで追求する組織だ。そして何より、代表である私自身が、今も毎日Next.jsとTypeScriptのコードを書き殴る超実践型のエンジニアだ。

私はすぐにPCを開き、開発環境を立ち上げた。そこから始まったのは、AIと私が完全にシンクロしてシステムをねじ伏せる、1時間の超高速トラブルシューティングだった。

## 経営者がコードを書き、自社AI『Antigravity』の設定を最適化する

原因はすぐに特定できた。共通の認可モジュールにおいて、特定のトークン失効時のリトライ処理が無限ループに入り、各APIのレートリミットを瞬時に食いつぶしていた。

これを1プロジェクトずつ手作業で修正し、ビルドを確認して、Vercelにマニュアルでデプロイしていくなど、あまりに非効率だ。私は自社で開発・運用しているAIエージェント制御ツール『Antigravity』のコンソールを開いた。

『Antigravity』は、AIパイプラインを自律稼働させるための心臓部だ。私はこのツールの設定ファイルを書き換え、エラーを検知したプロジェクトのトークンローテーション機能と、Gemini 2.5 Flashを活用したコード自動修正ロジックを割り込ませることにした。

Next.js（Version 16系）と、厳格なStrict Modeで制御されたTypeScriptのコードベースを『Antigravity』に読み込ませる。

「トークン失効時の指数バックオフ（Exponential Backoff）アルゴリズムを適用し、VercelのDeploy Hookを自動でキックする修正パッチを作成せよ」

私が指示を投げてから数十秒。画面には、型安全（Type-safe）が保証された美しいTypeScriptのコードが出力された。すぐにローカル環境で検証を実行する。エラーは出ない。

「よし、デプロイだ」

VercelのDeploy HookをAPI経由で一斉に叩く。10以上のプロジェクトのビルドプロセスが同時に走り出し、Vercelのダッシュボード上でグリーンの「Ready」マークが次々と点灯していく。

Slackのエラー通知が止まり、テスト投稿の成功ログが流れ始めた。

時計を見る。9時15分。
全滅した10以上のプロジェクトは、完全に息を吹き返した。発生からわずか1時間での完全復旧だった。

## 「綺麗事」の自動化はいらない。自分で傷を負い、自分で治す

私がこのエピソードで伝えたいのは、「私たちの技術力はすごい」という自慢話ではない。

世の中には「AIで完全自動化」「ノーコードで不労所得」といった、実体のない言葉が溢れている。しかし、システムを実際に運用したことがある人間なら誰でも知っているはずだ。APIの仕様変更、ライブラリの脆弱性、予期せぬエラー。自動化すればするほど、その牙城が崩れたときのインパクトは大きく、運用には泥臭い技術力と執念が必要になる。

私は、架空の数字や誇大広告で飾られたビジネスが大嫌いだ。だからこそ、FIでは「自分たちがヘビーユーザーとして使い倒し、検証した仕組み」しか世の中に提供しない。

現在、私たちは1人で11のサイトを自動運営し、累計1,500本以上の記事をAIで生成・公開し、Google Search ConsoleのデータをAPI経由で取得して自動リライトを回し続けている。

この膨大な「実証実験」のプロセスがあるからこそ、今回のようなAPIの突然の死にも、1時間でシステムを書き換えて復旧させるノウハウが蓄積される。自分で傷を負い、自分で治す。このサイクルを誰よりも速く回しているからこそ、私たちのプロダクトや受託開発には、他社が決して真似できない圧倒的な説得力がある。

元SESのエンジニアとしてキャリアをスタートした私は、仕様書通りのコードをただ書くだけの仕事に、ずっと息苦しさを感じていた。自分の書いたコードが、ビジネスにどう貢献しているのかが見えない開発はつまらない。

FIでは、書いたコードが、構築したAIパイプラインが、ダイレクトに事業を動かす。このスピード感と手応えは、一度味わうと病みつきになる。

## AIの限界を決めず、コードで未来の仕組みを実装したいあなたへ

私たちのチームに、お飾りのマネジメント層や、口先だけで動かない人間は必要ない。

私たちが求めているのは、技術の進化を誰よりも面白がり、自らキーボードを叩いてシステムを構築できるプレイヤーだ。Next.jsやTypeScriptの深い知識を持ち、LLM（GeminiやOpenAIなど）のAPIをただ叩くだけでなく、その限界や特性を理解してパイプラインを設計できるような、尖った技術者と一緒に働きたい。

FIはフルリモートワークを導入しており、無駄な社内ミーティングや、形だけのレポート作成は一切排除している。やるべきことはシンプルだ。自律的に動き、圧倒的なスピードで仮説を検証し、システムを本質的に改善し続けること。

「AIに使われる開発者」になるのか。それとも、「AIを最高の相棒にして、1人で10倍、100倍の価値を生み出す開発者」になるのか。

私たちは、後者の人間が集まる場所であり続けたい。

朝一番のトラブルを、最高のバイブスで、コードとAIを使って1時間で解決する。そんな圧倒的なスピード感の中で、自分の技術力を極限まで試してみたいと思うエンジニアからの挑戦を、私は待っている。

コードで語り、システムで証明する。それだけだ。
