> 🤖 この記事は **2026-07-02** にFuturistic Imaginationの自動コンテンツパイプライン（Gemini 2.5 Flash）によって生成されました。

朝7時、ベッドの脇でiPhoneが激しく振動した。

SlackとDiscordからのエラー通知が止まらない。画面を見ると、自社で運営している11のオウンドメディア、そしてクライアント向けに提供しているGemini API搭載のコンテンツ自動生成パイプラインのステータスが、見事なまでに真っ赤に染まっていた。

「主要なAPI連携が、すべて死んでいる」

原因はすぐに特定できた。Buffer（SNS投稿自動化ツール）とYouTube APIのアクセストークンの一斉切れだ。認証仕様の変更に伴い、バックエンドで動いていたトークンの自動リフレッシュ処理が失敗。結果として、連携している10個のプロダクトが一斉に停止したのだ。

元SESであり、今はAIを駆使して「極限の自動化」を追求するプレイングエントレプレナーとして、私の血が騒いだ。

Next.jsとTypeScript（Strict Mode）で構築された各Webアプリケーション、そしてVercel上で稼働するサーバーレス関数。これら10プロダクトのソースコードを1つずつ開き、手作業でトークンを再設定し、デプロイし直すのか。そんな不毛な手作業をしていたら、貴重な1日が丸ごと潰れてしまう。

「同じ手作業を二度繰り返さない」

これが私の絶対的なルールだ。私は即座に、自社で開発しドッグフーディング（自社実証）を続けている自律型AIエージェント『Antigravity』を立ち上げた。

---

## 自律AIの承認設定を『Unrestricted』にする瞬間

通常、AIエージェントにコードの書き換えやデプロイを任せる際、予期せぬ挙動を防ぐために、実行の節目で人間の「承認（Approval）」を必要とする設定にしている。しかし、1分1秒を争うこの状況下で、いちいちボタンをクリックしている余裕はない。

私は『Antigravity』の承認設定を『Unrestricted（制限なし）』に切り替えた。

これは、AIに自社の全コードベースへの書き込み権限と、Vercelのデプロイ権限を完全に委ねることを意味する。もちろん、リスクはある。だが、自社で作り込んだシステムと、Gemini APIの性能を私は信頼していた。

「タスク：BufferおよびYouTube APIの新しい認証ロジックを実装し、全10プロダクトの環境変数を更新、Vercelへデプロイせよ。TypeScriptの型エラーは一切許容しない」

プロンプトを入力した瞬間、画面上で無数のコンソールが動き出した。

AIエージェントが、各プロダクトのGitHubリポジトリを次々とチェックアウトしていく。ESM（ECMAScript Modules）で記述されたNode.jsの監視・投稿スクリプトを解析し、問題となっているトークン失効時のフォールバック処理の不備を発見した。

AIは自ら修正コードを書き、`npm run build` を走らせて型安全性を確認する。TypeScriptのStrict Modeを有効にしているため、型定義に少しでも矛盾があればビルドは容赦なく落ちる。AIはエラーを吐くたびに「自らエラーログを読み込み、自律的にコードを修正する」というループを、秒速で回し続けた。

---

## 泥臭いドッグフーディングからしか、本物のプロダクトは生まれない

私が画面で見守る中、AIエージェントはバグを修正したコードをGitHubにプッシュし、VercelのDeploy Hookを叩いていく。

「弊社自身が使っている状態を作らないと、顧客には刺さらない」

これが私の強い信念だ。今回のように、自社メディア11サイトを1人で運営し、累計1,500本以上のAI記事を生成・管理する中で発生する「生々しいトラブル」こそが、自社プロダクトを研ぎ澄ます最高のスパイスになる。

APIエラーの裏側で、私はもう一つの問題に気づいた。検索順位の低い記事をAPI経由で自動検出し、自動でリライトしてIndexNowで送信する「GSC/SEO自動改善パイプライン」も、このエラーの影響で一部処理がスタックしていたのだ。

私はAntigravityに追加の指示を出した。
「エラー検知時、処理を単に落とすのではなく、Discordへ即座に詳細なペイロード付きで通知し、影響のない他の記事のリライト処理は継続するようにフォールバックを実装しろ」

AIは即座に応答し、例外処理をコードに組み込んでいった。

トラブル発生からわずか2時間。手作業であれば1日はかかったであろう「10プロダクトのコード修正、環境変数更新、本番デプロイ、そして例外処理の実装」が、完全に完了した。すべてのシグナルが緑色に戻り、オウンドメディアは再び自動で自律稼働を始めた。

## 圧倒的なスピードと「仕組み化」に執念を燃やす仲間へ

私は、スマートで綺麗でお行儀の良い開発がしたいわけではない。

市場の技術進化は速い。Gemini APIも2.0から2.5 Flashへと瞬く間にアップデートされ、それに対応する開発スピードが求められる。そこで理想論や綺麗な設計に固執して立ち止まるくらいなら、AIをフル活用して「その日のうちにコードを書き換え、デプロイまで完了させる」フットワークの軽さを持っていたい。

Futuristic Imagination LLCが求めているのは、こうした「泥臭い自動化」を最高に面白いと思えるエンジニアだ。

私たちは、AIエージェントを文字通り「部下やパートナー」として扱い、限界まで省力化した環境で勝負している。今回のようなAPI障害すらも、自社ツールの性能を極限まで高めるための「格好のテストケース」として捉えられるような、タフで知的な好奇心を持つ人と働きたい。

用意された仕様書通りにコードを書く仕事は、ここにはない。あるのは、リアルな事業課題と、それをテクノロジーで爆速で解決する圧倒的な裁量だけだ。

最先端のAI技術を、泥臭く、しかし誰よりもエレガントに仕組み化していく。このプロセスに心躍るエンジニアからの挑戦を、私は待っている。

AIを使い倒しているこの会社が、最も人間らしい働き方をしている。
