こんにちは、Futuristic Imagination LLC 代表の佐藤琢也です。

今日のこの採用ブログ記事、実はあなたがお読みになっているこの文章自体が、私たちが日々取り組んでいるAIと自動化の最先端の挑戦なんです。正直な話、大手企業が発信するような「完璧な採用ブログ」とは一線を画しています。なぜなら、これは私自身の思考と、Gemini API、そして自動生成された私のペルソナ情報が協業して生み出された、ある種の実験だからです。

元々SESからAI起業という道を歩んできた私が、なぜこんなにもAIと自動化に執着するのか？それは、手作業の繰り返しに追われることなく、本当に価値ある創造的な仕事に集中したいという強い思いがあるからです。そして、その思想をこの採用活動にも徹底的に持ち込んでいます。このブログ記事は、AIが私のペルソナプロファイルを学習し、今日のトピックに合わせて生成したドラフトを、私が最終的に調整・加筆したものです。

あなたがもし、「この会社で働いてみたい」と感じてくれたなら、それは私たちのAIが私の思考を的確に捉え、私が伝えたい熱量を記事に反映できた証拠。そして、何よりもこの挑戦を通して、FIがどんな価値観を持ち、どんな未来を創ろうとしているのかを、肌で感じてほしいと思っています。

## AIで「採用コンテンツ」を自動化する。その挑戦の裏側

採用活動といえば、企業の顔となる重要な業務。だからこそ、多くの会社が人力で、言葉を選び、丁寧なコンテンツを作ります。ですが、Futuristic Imaginationでは、この採用コンテンツですらAIによる自動化の可能性を追求しています。

「AIが書いた採用記事なんて、本当に気持ちが伝わるのか？」そう思われるかもしれませんね。正直に言うと、私も最初はそのジレンマと向き合いました。しかし、私の結論は「人間が持つべきは、AIにはできない共感や熱意、そして最終的な判断だ」というものです。

今回のこのブログ記事も、私の過去の言動や思考パターンをまとめたペルソナプロファイルをGemini APIに学習させ、今日のテーマに沿って「佐藤琢也らしい言葉」で記事を生成する試みです。Vercel CronとGitHub Actionsを組み合わせたパイプラインで、記事の生成から公開までを可能な限り自動化しています。これ自体が、私たちがお客様に提供している「AIオウンドメディア構築」や「Gemini API コンテンツ生成パイプライン」の、まさにライブデモンストレーションなんです。

もちろん、すべてがスムーズに進むわけではありません。実は、AIが生成した記事が私の意図と少しずれたり、あるいはTypeScriptのStrict Modeで型エラーが出たり、Geminiの出力フォーマットが崩れたり……と、試行錯誤の連続です。その度にsession_log.mdに「バグの原因→解決策→今後の注意点」を詳細に記録し、次に活かしています。失敗を隠さず、むしろ学びの糧として公開することで、私たちの技術への真摯な姿勢と、信頼につながると信じています。

私たちは、採用活動においても「繰り返しは全て自動化し、人間の判断が必要な部分だけに集中する」という哲学を貫いています。AIは単なるツールではありません。最高のパートナーとして、私たちの創造性と効率を最大限に引き出す存在なんです。

## 「実証主義」が拓く未来。NDAと情報の壁を越えて

Futuristic Imaginationの核となる価値観は「実証済みであること」。私は、自分が実際に動かしているシステムだけを提案し、試していないものは売らないと決めています。お客様への提供価値の根源は、この「実証主義」に他なりません。

しかし、B2B受託開発、特に大手放送業界のようなクライアントとのAI開発支援では、機密性の高いNDA（秘密保持契約）がつきものです。正直な話、具体的な案件内容やお客様の名前をオープンにできないのは、私たちのような実績ベースの会社にとっては歯がゆい部分です。

だからこそ、私たちは情報の伝え方、信頼の築き方に工夫を凝らしています。今回の「最新のペルソナ統合」も、その一環なんです。私のペルソナプロファイルに「自動生成日: 2026-04-22 16:10 JST」と明記されていることにお気づきでしょうか？これは、私の思考や活動ログが常にAIによって分析・更新されていることを示唆しています。NDAで開示できない具体的な案件内容に代わり、「佐藤琢也がどのように思考し、どのようにシステムを動かしているか」を間接的に伝える試みなんです。

このペルソナ情報をAIに学習させる際も、NDAに抵触しない範囲で、かつFIの技術力や哲学を最大限に伝えるよう、細心の注意を払って設計しています。法的なコンプライアンスは私たちのビジネスの基盤であり、特商法・プライバシーポリシー・NDA遵守は徹底しています。

私たちは、失敗も包み隠さず記録・公開することで、読者やクライアントからの信頼を得てきました。session_log.mdは、その正直さの象徴です。ISRキャッシュのエラー、Geminiの出力フォーマット崩れ、TypeScript Strict Modeの罠など、私たちが実際に踏んだ地雷を正直に語ることで、単なる成功談ではない、リアルな技術的知見と人間としての信頼を築いているんです。この透明性こそが、私たちが目指す「実証主義」の真髄であり、FIの強みだと自負しています。

## 1人でも「11サイト」自動運用。その先に描くFIの未来

「1人でも11サイトを自動運用」。このフレーズを聞いて、驚かれる方もいるかもしれませんね。正直、普通に考えたら不可能です。しかし、Futuristic Imaginationではこれを実現しています。

なぜこれが可能なのか？それは、私が「自動化への執着」と呼ぶほどの徹底した効率追求があるからです。Vercel Cron、GitHub Actions、Gemini APIを組み合わせたパイプラインで、記事生成・SNS投稿・レポーティングをほぼ全自動化。Next.jsやTypeScriptといった最新の技術スタックを駆使し、LinkedIn・Qiita・Zenn・Note・Xへの全自動投稿システムも構築しています。複数のリポジトリ・システムを並行管理しながら、それぞれの状態を常に正確に把握する。このスピーディーかつ実証主義的な仕事スタイルが、この圧倒的な生産性を生み出しています。

これは単なる「手抜き」ではありません。むしろ逆です。繰り返し作業はAIに任せ、人間の判断が必要な「創造性」や「戦略立案」に、私の全リソースを集中させるための選択なんです。AIは、私たちの想像力を現実にするための最高のツールであり、パートナーです。

FIが目指すのは、AIを駆使して、圧倒的な生産性と質の高さを両立する未来です。私たちが得意とする4つの領域、つまり「AIオウンドメディア構築（0→1）」「WordPress → Next.js 移行 + 自動化」「Gemini API コンテンツ生成パイプライン」「SNS・Discord 自動通知連携」は、その未来を実現するための柱です。

このような環境で働くことは、まさに最先端のAI自動化技術を「実践」することに他なりません。理論より実装を優先し、問題が発生すれば即座に根本原因を特定して解決する。そんなスピーディーな環境で、新しい技術に臆することなく飛び込み、自ら手を動かし、結果を出す。そんな挑戦を求める人にこそ、Futuristic Imaginationは最高の舞台を提供できると信じています。
